# Day 13 – AWS Configuration Compliance Monitoring Lab

## Objective

Implement configuration monitoring using AWS Config.

## Steps Performed

1. Logged into AWS console
2. Enabled AWS Config
3. Configured resource recording
4. Created S3 bucket for configuration history
5. Added compliance rule for public S3 buckets

## Security Benefits

AWS Config continuously monitors cloud resource configurations.

It helps detect misconfigurations and enforce compliance policies.

## Compliance Mapping

Control Type: Detective Control

ISO 27001 – Configuration Management
NIST – Configuration Monitoring
SOC 2 – System Security Controls
