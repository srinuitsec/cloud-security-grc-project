# Day 8 – AWS GuardDuty Threat Detection Lab

## Objective

Enable AWS GuardDuty to detect security threats in the AWS environment.

## Steps Performed

1. Logged into AWS console
2. Opened GuardDuty service
3. Enabled GuardDuty monitoring
4. Reviewed threat detection findings

## Monitoring Sources

CloudTrail logs
DNS logs
VPC Flow logs

## Security Benefits

GuardDuty provides automated threat detection using machine learning and threat intelligence.

It detects suspicious activities such as unauthorized access, reconnaissance attempts, and compromised credentials.

## Compliance Mapping

Control Type: Detective Control
