# Day 9 – AWS Encryption and Data Protection Lab

## Objective

Implement encryption using AWS Key Management Service (KMS).

## Steps Performed

1. Logged into AWS console
2. Created encryption key (grc-encryption-key)
3. Created S3 bucket with KMS encryption enabled
4. Uploaded file and verified encryption status

## Security Benefits

Encryption protects sensitive data stored in cloud resources.

It ensures that only authorized users can decrypt and access the data.

## Compliance Mapping

Control Type: Preventive Control

GDPR – Data Protection
